# 大模型开发工程师岗位画像（转写）

## 岗位定位与职责边界

- 负责大模型工程化的实现、封装与平台集成，连接底层算法研究与上层应用系统。
- 将大模型能力封装为可调用接口、模块化组件与服务化架构，确保在业务中高效、安全、稳定运行。
- 在模型全生命周期中承担推理服务搭建、API 封装、前后端调用、缓存机制设计、任务调度、多模型集成与评估体系建设等工程任务。

## 关键技术能力要求

1. **大模型调用与封装能力**
   - 使用主流框架/接口（如 HuggingFace Transformers、OpenAI API、ChatGLM、Qwen）加载并调用模型。
   - 熟悉 Prompt 构造、Token 控制、输出解析等工程细节。

2. **推理服务部署与优化经验**
   - 能完成本地、云端与容器化部署，掌握 Docker、Conda、Kubernetes 等工具。
   - 熟悉推理性能优化（如 KV Cache、批处理、并发控制）。

3. **系统架构与微服务设计能力**
   - 理解 Web 系统与微服务设计理念，能做模块划分、服务拆分、任务路由与模块协同。
   - 具备 RESTful 接口设计、模块解耦、异常处理、日志记录等系统级能力。

4. **任务调度与控制逻辑构建**
   - 构建智能体或多模块系统中的 Prompt 调用状态机、上下文管理、历史信息追踪与决策逻辑。
   - 支持多轮交互、插件调用、RAG 融合等功能并保证可扩展性。

5. **工程代码质量与可维护性**
   - 强调代码组织、测试覆盖、日志记录与错误处理能力。
   - 对模型调用链路进行模块化与抽象化，提升系统可扩展性与稳定性。

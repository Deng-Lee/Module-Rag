## 二、参考答案

### <a id="a-mcp协议"></a>Q1 参考答案：MCP 起什么作用，为什么不直接暴露 REST API？

MCP 的核心价值不是“抹平不同模型厂商的 REST API 差异”，而是**定义 AI Client 与工具/数据源之间的统一协议层**。在这个项目里，MCP Server 把知识库能力封装成稳定的 tools，例如 `library_ingest`、`library_query`、`library_query_assets`、`library_get_document` 等，上层 Agent 或 Copilot 通过 JSON-RPC / stdio 直接调用。

相比只暴露 REST API，MCP 的优势是：

1. **面向 Agent 生态的标准化工具协议**
2. **输入输出 schema 清晰，便于模型选择工具**
3. **无需为每个上层 AI Client 单独适配**
4. **可直接承载引用透明性、结构化返回、trace_id 等字段**

所以正确回答应强调：REST API 是服务暴露方式，MCP 是 Agent 工具集成协议，两者不是简单替代关系。

### <a id="a-量化指标"></a>Q2 参考答案：5 分钟降到 1 分钟内怎么测？

这类量化结果如果要站得住，至少应说明：

1. **baseline 是什么**
   - 原系统按关键词人工检索多份资料
2. **测量对象是什么**
   - 同一批问题、同一批文档、同一批使用者场景
3. **测量方式是什么**
   - 人工任务计时 + 系统 trace 打点
4. **变量控制了什么**
   - 文档集、embedding 模型、chunk 策略、top_k、过滤条件等保持不变
5. **样本量是多少**
   - 至少多少条 query / 多少轮任务

你这轮回答的问题是：
- `5 分钟` 主要还是经验值，不够硬
- `1 分钟内` 的 trace 时间和人工查找时间不是完全同一口径
- baseline 的描述不够严谨，后半段又切到了 rerank 对比，和“人工检索 vs 系统检索”这条主线混在一起

### <a id="a-cross-encoder"></a>Q3 参考答案：Cross-Encoder 与 Bi-Encoder 的区别

标准回答应包括：

1. **Bi-Encoder**
   - Query 和 Document 分别编码
   - Document 向量可离线预计算
   - 适合大规模粗排召回
2. **Cross-Encoder**
   - Query 和 Candidate 拼接后联合输入模型
   - 每个 `(query, candidate)` 都要在线推理
   - 能捕捉更强交互特征，精度高但成本高
3. **为什么放在 rerank 层**
   - 因为只适合对 Top-N 小候选集打分
4. **当前系统里的主输入**
   - 优先 `(query, chunk_retrieval_text)`，缺失时回退 `chunk_text`
5. **代价**
   - 候选集越大，延迟和资源消耗越高

你这轮答对了大方向，但有两个问题：
- 把 Cross-Encoder 说成“拼成一个向量”不够准确
- 没提到你们当前系统里具体用的输入对象是 `chunk_retrieval_text`

## 三、简历包装点评

### 包装合理 ✅

- **“混合检索 + Cross-Encoder 重排方案延迟较高但精度最佳”**：你能说出 Cross-Encoder 只适合精排、小候选集成本高，方向是对的。
- **“流程里有很多打点”**：你能自然把 trace 打点和时间统计联系起来，说明你对系统可观测性有真实接触。

### 露馅点 ❌

- **“理解 MCP 协议”** → 你把 MCP 的重点答成了“适配不同模型厂商的 API 差异”，这不是这个项目里 MCP 的核心定位。**严重性：中**
- **“量化结果可信”** → 你承认 `5 分钟` 是经验性数字，这会直接削弱简历里这条指标的可信度。**严重性：中**
- **“Cross-Encoder 实现理解”** → 大方向正确，但措辞不够精确，没有结合当前项目真实输入口径。**严重性：低**

### 改进建议

- MCP 这题以后固定答法：`统一 Agent 工具调用协议 + 标准化 tool schema + stdio/JSON-RPC`。
- 对简历中的时间收益数据，准备一个更规范的口径，比如：
  - 人工流程计时样本数
  - 系统 trace 口径
  - 两者如何对比
- Cross-Encoder 相关题尽量加入项目化描述：`fusion 后 Top-N`、`(query, chunk_retrieval_text)`、`rerank_latency_ms`。

  1. MCP 在这个项目中的真实作用
